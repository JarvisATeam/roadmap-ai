# roadmap.ai — Strategy Blueprint

## Market Position

**Core insight:** Scale solved capability. Continuity is now the dominant failure mode in deployed AI systems.

**White space:** ChatGPT Projects, Claude Projects, Notion AI all have memory — none have continuity as primary design objective.

## The Unique Shift

- Typical AI memory: "what was said"
- Continuity engine: "what this means for next action"

## One Engine, Three Modes

    CONTINUITY ENGINE (shared)
    ├── Learn Mode (students/self-learners)
    ├── Build Mode (founders/developers) ← START HERE
    └── Research Mode (researchers/analysts)

Same backend. Different interfaces. Different language/depth/evaluation.

## Moat

1. **Continuity over time** - weeks/months, not just sessions
2. **Thread recovery** - "Git blame for your thinking"
3. **Explainable AI** - score + source + confidence + "what changed"
4. **Context as moat** - compounds over time, can't be replicated quickly

## MVP (30 days)

**Week 1:** Validate pain (5-10 interviews)
**Week 2:** Core objects (Mission/Milestone/Step/Decision)
**Week 3:** Re-entry + plan-diff engine
**Week 4:** Export + onboarding

**Success metric:** One user makes a real prioritization decision based on re-entry brief.

## Positioning

**Primary wedge:** Solo founders/builders/makers

**Copy to test:**
- "Never lose the thread again."
- "Your project remembers everything. You just have to build."

**Category name:** Threadkeeping AI

## What We DON'T Build in MVP

- ❌ Finetuned model
- ❌ Gantt/timeline UI
- ❌ 10+ integrations
- ❌ Autonomous PM agent
- ❌ Learn/Research modes (Build mode only)

## Long-term Vision

After MVP validation, one engine scales to all modes. Realistic sequence: Build → Learn → Research.

**Mission:** "Follow humans and projects from idea to goal" — whether it's 6th grade math or $1M SaaS.
